# Local-first Paper Highlights

Reference: [Ink & Switch - Local-first software](https://www.inkandswitch.com/essay/local-first/).

Local latency changes product tone. Fast, resilient interactions feel less theatrical and more dependable. The paper's strongest contribution is not just the technical architecture; it is the argument that software feels more humane when the user's work remains available and responsive without asking permission from the network.

Highlights worth carrying forward:

- Local-first systems preserve ownership without forcing users to think like backup administrators.
- Sync should be visible but not demanding.
- Collaboration improves when the local copy is the default home of the work, not a cache of some remote original.

This source keeps sharpening [[Local First Principles]], [[Vault Layout]], and the product's calmer trust story.
